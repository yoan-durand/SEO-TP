[modifier] généralités les ordinateurs furent abor utilisés pour calcul (en nombres entiers abord puis flottants). peut cependant les assimiler simples calculateurs : effet, résultat traitement ordinateur peut être non seulement une série nombres, mais aussi nouveau programme (utilisable par cet ordinateur par autre). dans architectur von neumann, les données sont banalisées peuvent être interprétées indifféremment comme des nombres, des instructions, des valeurs logiques tout symbole défini arbitrairement (lettre l’alphabet, par exemple). calcul représente une des applications possibles. dans cas, les données sont traitées comme des nombres. l’ordinateur est utilisé aussi pour ses possibilités organisatio l’information, entre autres sur des périphériques stockage magnétique. calculé fin des années 1980 que sans les ordinateurs faudrait toute population française juste pour faire dans pays seul travail des banques. cette capacité d’organiser les informations généralisé l’usage traitement texte dans grand public ; gestion des bases données relationnelles permet également retrouver consolider des informations réparties vues par utilisateu comme plusieurs tables indépendantes. cette création néologisme fut origin traductions multiples des expressions supercomputer, superordinateur supercalculateur, quantum computer, calculateur quantique ordinateur quantique. dans dernier cas, utilisatio mot « ordinateur » est justement surfaite car les possibilités envisageables pour calcul quantique sont loin polyvalence « ordinateur ». l’expérience appris distinguer dans ordinateur deux aspects, dont second avait été départ sous-estimé : l’architecture physique, matérielle (alias hardware hard) ; l’architecture logicielle (alias software soft) ; ordinateur très avancé techniquement pour son époque comme gamma compagnie bull n’eut pas succès attendu, pour simple raison qu’il existait peu moyens mettre œuvre commodément ses possibilités techniques. logiciel son complément les services (formation, maintenance, ...) forme depuis milieu des années 1980 l’essentiel des coûts d’équipement informatique, matériel n’y ayant qu’une part minoritaire. les ordinateurs peuvent être sensibles aux bombes iem. [modifier] fonctionnement vue ensembl des différents organes ordinateur parmi toutes les machines inventées par homme ordinateu est celle qui rapproche plus concept anthropologique suivant : organe entrée organe traitement informatio organe sortie. chez homme les organes entré sont les cinq sens, organ traitement est cerveau dont les logiciels sont apprentissag avec des mises jour constantes cours vie, puis les organes sortie sont les muscles. pour les ordinateurs modernes, les organes entré sont clavier souris les organes sortie, écran imprimante graveur dvd, etc.] les techniques utilisées pour fabriquer ces machines ont énormément changé depuis les années 1940 sont devenues une technologie (c’est-à-dire ensemble industriel organisé autour techniques) part entière depuis les années 1970. beaucoup utilisent encore les concepts définis par john von neumann, bien que cette architecture soit régression : les programmes modifient plus guère eux-mêmes (ce qui serait considéré comme une mauvaise pratique programmation), matériel prend compte cette nouvelle donne séparant nettement stockage des instructions des données, compris dans les caches. l’architecture von neumann décomposait l’ordinateur quatre parties distinctes : l’unité arithmétique logique (ual) unité traitement : son rôle est d’effectuer les opérations base, peu comme ferait une calculatrice ; l’unité contrôle. c’est l’équivalent des doigts qui actionneraient calculatrice ; mémoire qui contient fois les données programme qui dira l’unité contrôle quels calculs faire sur ces données. mémoire divise entre mémoire vive (programmes données cours fonctionnement) mémoire permanente (programmes données base machine) ; les entrées-sorties : dispositifs qui permettent communiquer avec monde extérieur. [modifier] ual l’unité arithmétique logique ual est l’élément qui réalise les opérations élémentaires (additions, soustractions...), les opérateurs logiques (et, ou, ni, etc.) les opérations comparaison (par exemple comparaison d’égalité entre deux zones mémoire). c’est l’ual qui effectue les calculs l’ordinateur. l’unité contrôle prend ses instructions dans mémoire. celles-ci lui indiquent qu’elle doit ordonner l’ual et, comment elle devra éventuellement agir selon les résultats que celle-ci lui fournira. une fois l’opération terminée, l’unité contrôle passe soit l’instruction suivante, soit une autre instruction laquelle programme lui ordonne brancher. unit contrôle facilite communication entre unit arithmétique logique, mémoire ainsi que les périphériques. elle gère plupart des exécutions des instructions dans ordinateur [modifier] mémoire sein système, mémoire peut être décrite comme une suite cellules numérotées contenant chacune une petite quantité d’informations. cette information peut servir indiquer l’ordinateur qu’il doit faire (instructions) contenir des données traiter. dans plupart des architectures, même mémoire qui est utilisée pour les deux fonctions. dans les calculateurs massivement parallèles, admet même que des instructions programmes soient substituées d’autres cours d’opération lorsque cela traduit par une plus grande efficacité. cette pratique était jadis courante, mais les impératifs lisibilité génie logiciel fait régresser, hormis dans cas particulier, depuis plusieurs décennies. cette mémoire peut être réécrite autant fois que nécessaire. taille chacun des blocs mémoire ainsi que technologie utilisée ont varié selon les coûts les besoins : bits pour les télécommunications, bits pour l’instrumentation (dec) bits pour gros calculateurs scientifiques (control data). consensus fini par être trouvé autour l’octet comme unité adressable d’instructions sur format 8 octets. dans tous les cas figure, octe reste adressable, qui simplifie écritur des programmes. les techniques utilisées pour réalisation des mémoires ont compris des relais électromécaniques, des tubes mercure sein desquels étaient générées des ondes acoustiques, des transistors individuels, des tores ferrite enfin des circuits intégrés incluant des millions transistors. [modifier] entrées-sorties les dispositifs d’entrée/sortie permettent l’ordinateur communiquer avec l’extérieur. ces dispositifs sont très importants, clavier l’écran. carte réseau permet par exemple relier les ordinateurs réseau informatique, dont plus grand est internet. point commun entre tous les périphériques d’entrée est qu’ils convertissent l’information qu’ils récupèrent l’extérieur données compréhensibles par l’ordinateur. l’inverse, les périphériques sortie décodent l’information fournie par l’ordinateur afin rendre compréhensible par l’utilisateur. [modifier] bus ces différentes parties sont reliées par trois bus, bus adresse bus données bus commande. bus est groupement certain nombre fils électriques réalisant une liaison pour transporter des informations binaires codées sur plusieurs bits. bus adress transporte les adresses générées par (unité centrale traitement) pour sélectionner une case mémoire registre interne des blocs. nombre bits véhiculés par bus dépend quantité mémoire qui doit être adressée. bus données transporte les données échangées entre les différents éléments système. bus contrôle transporte les différents signaux synchronisation nécessaires fonctionnement système : signal lecture (rd), signal écritur (wr), signal sélection (cs : chip select). [modifier] architecture miniaturisation permet d’intégrer l’ual l’unité contrôle sein d’un même circuit intégré connu sous nom microprocesseur. typiquement, mémoire est située sur des circuits intégrés proches processeur, une partie cette mémoire, mémoire cache, pouvant être située sur même circuit intégré que l’ual. l’ensemble est, sur plupart des architectures, complété d’une horloge qui cadence processeur. bien sûr, souhaite ell soit plus rapide possible, mais peut pas augmenter sans limites vitesse pour deux raisons : plus l’horloge est rapide plus processeur dégage chaleur (selon carré fréquence). une trop grande température peut détériorer processeur ; existe une cadence processeur devient instable ; génère des erreurs qui mènent plus souvent plantage. tendance été partir 2004 regrouper plusieurs ual dans même processeur, voire plusieurs processeurs dans même puce. effet, miniaturisation progressive (voir loi moore) permet sans grand changement coût. une autre tendance, depuis 2006 chez arm, est aux microprocesseurs sans horloge : moitié dissipation thermique est effet due aux signaux horlog quand microprocesseur fonctionne ; plus, microprocesseur sans horloge une consommation presque nulle quand fonctionne pas : seul signal horlog nécessaire est alors celui destiné rafraîchissement des mémoires. cet atout est important pour les modèles portables. principal écart fonctionnel aujourd’hui par rapport modèle von neumann est présence sur certaines architectures deux antémémoires différentes : une pour les instructions une pour les données (alors que modèle von neumann spécifiait une mémoire commune pour les deux). raison cet écart est que modification par programme ses propres instructions est aujourd’hui considérée (sauf sur les machines hautement parallèles) comme une pratique proscrire. dès lors, contenu cache données doit être récrit mémoire principale quand est modifié, sait que celui cache d’instructions n’aura jamais l’être, d’où simplification des circuits gain performance. [modifier] instructions les instructions que l’ordinateur peut comprendre sont pas celles langage humain. matériel sait juste exécuter nombre limité d’instructions bien définies. des instructions typiques comprises par ordinateur sont « copier contenu cellule 123 placer dans cellule 456 », « ajouter contenu cellule 321 celui cellule 654 placer résultat dans cellule 777 » « si contenu cellule 999 vaut exécuter l’instruction cellule 345 ». mais plupart des instructions composent deux zones : l’une indiquant quoi faire, nommée code opération, l’autre indiquant faire, nommée opérande. sein l’ordinateur, les instructions correspondent des codes code pour une copie étant par exemple 001. l’ensemble d’instructions qu’un ordinateur supporte nomme son langage machine, langage qui est une succession chiffres binaires, car les instructions données qui sont comprises par processeur (cpu) sont constituées uniquement (zéro) (un). courant électrique passe pas. courant électrique passe. général, les programmeurs n’utilisent plus type langage, mais passent par que l’on appelle langage haut niveau qui est ensuite transformé langage binaire par programme spécial (interpréteur compilateur selon les besoins). les programmes ainsi obtenus sont des programmes compilés compréhensibles par ordinateu dans son langage natif. certains langages programmation, comme l’assembleur sont dits langages bas niveau car les instructions qu’ils utilisent sont très proches celles l’ordinateur. les programmes écrits dans ces langages sont ainsi très dépendants plate-forme pour laquelle ils ont été développés. langage beaucoup plus facile relire que l’assembleur, permet donc aux programmeurs d’être plus productifs. pour cette raison, l’a plus plus utilisé mesure que les coûts matériel diminuaient que les salaires horaires des programmeurs augmentaient. [modifier] logiciels les logiciels informatiques sont longues listes d’instructions exécutables par ordinateur. nombreux programmes contiennent des millions d’instructions, effectuées pour certaines manière répétitive. nos jours, ordinateur personnel exécute plusieurs milliards d’instructions par seconde. depuis milieu des années 1960, des ordinateurs exécutent plusieurs programmes simultanément. cette possibilité est appelée multitâche. c’est cas tous les ordinateurs aujourd’hui[quand ?]. réalité, chaque cœur processeur n’exécute qu’un programme fois, passant d’un programme l’autre chaque fois que nécessaire. rapidité processeur est suffisamment grande par rapport nombre tâches exécuter, l’utilisateur aura l’impression d’une exécution simultanée des programmes. les priorités associées aux différents programmes sont, général, gérées par système exploitation [modifier] système d’exploitation article détaillé : système exploitation système d’exploitation est programme central qui contient les programmes base nécessaires bon fonctionnement des applications l’ordinateur. système d’exploitation alloue les ressources physiques l’ordinateur (temps processeur, mémoire...) aux différents programmes cours d’exécution. fournit aussi des outils aux logiciels (comme les pilotes) afin leur faciliter l’utilisation des différents périphériques sans avoir connaître les détails physiques. wikipédia avertissements version mobile ordinateur article wikipédia, encyclopédi libre. aller navigation, rechercher ordinateur est une machine électronique qui fonctionne par lecture séquentielle ensemble instruction qui lui font exécuter des opérations logiques arithmétiques sur des chiffres binaires. dès mise sous tension, ordinateur exécute, après autre des instructions qui lui font lire, manipuler, puis réécrire ensemble données auquel accès. des tests des sauts conditionnels permettent changer instructio suivante, donc agi différemment fonction des données des nécessités moment. les données manipuler sont obtenues, soit par lecture mémoires, soit par lecture composants interfac (périphériques) qui représentent des données physiques extérieures valeurs binaires (déplacement souris, touche appuyée sur clavier, température, vitesse, compression...). une fois utilisées, manipulées, les données sont réécrites, soit dans des mémoires, soit dans des composants qui peuvent transformer une valeur binaire une action physique (écriture sur une imprimante sur moniteur, accélération freinage véhicule, changement température four ...). ordinateu peut aussi répondre des interruptions qui lui permettent d’exécuter des programmes réponses spécifiques chacune, puis reprendre l’exécution séquentielle programme interrompu. technique actuelle des ordinateurs date milieu xxe siècle. ils peuvent être classés selon plusieurs critères[1] (domaine application taille architecture). éclaté ordinateur personnel : écran carte mère processeur parallel ata mémoire vive (ram) ordinateur apparait dans les dictionnaires xixe siècle comme synonyme peu usuel ordonnateur: celui qui met ordre[2]. puis locution ordinateur électronique désigne 1960 une machine qui lit classe sans intervention humaine[3]. mot français semble avoir été introduit par ibm dans les années 60[4], pour éviter mot computer français. mot anglais computer vient ancie français, verbe computer [5]. [modifier] historique imag plus célèbre début histoir informatique[6 portrait jacquard, tissé soie sur métier jacquard, demandait lecture 24 000 cartes plus 1 000 trous chacune (1839). étai produit que sur demande. charles babbage utilis souvent pour expliquer ses idées sur que fut première esquisse ordinateur, machine analytique, qui utilisait des cartes jacquard pour ses commandes ses données[7] article détaillé : histoire des ordinateurs. [modifier] machines calculer les machines calculer jouèrent rôle primordial dans développement des ordinateurs pour deux raisons tout fait indépendantes. part, pour leurs origines : pendant développement machine calculer automatique imprimante 1834 charles babbage commença imaginer machine analytique, l’ancêtre des ordinateurs. c’était une machine calculer programmée par lecture cartes perforées (inspirées métier jacquard), avec lecteur cartes pour les données pour les programmes, avec des mémoires, calculateur central des imprimantes qui inspirera développement des premiers ordinateurs cent ans plus tard (howard aiken fit beaucoup référence aux machines babbage quand convainquit ibm construire harvar mark 1937) ; qui nous amènera aux mainframes des années 1960. autr part, leur propagation fit grâce commercialisation 1971 premier microprocesseur, inte 4004, qui fut inventé pendant développement machine calculer électronique pour compagnie japonaise busicom, qui est origin explosio micro-informatique partir 1975[8] qui réside cœur tous les ordinateurs actuels quel que soit leurs tailles fonctions (bien que seulement 2 % des microprocesseurs produits chaque années soient utilisés comme unités centrales ordinateur les 98 % restant sont utilisés dans construction voitures, robots ménager, montres, caméras surveillance[9]...). [modifier] électromécanique mécanographie outre les avancées observées dans industri textile celles électronique les avancées mécanographie fin xixe siècle, pour achever les recensements aux états-unis amérique mécanisation cryptographie début xxe siècle, pour chiffrer puis déchiffrer automatiquement des messages, développement des réseaux téléphoniques base relais électromécaniques), sont aussi prendre compte pour comprendre avènemen nouveau genre machine qui calculent pas (comme font/faisaient les calculatrices), mais lisent interprètent des programmes qui -eux- calculent. pour monde des idées, avant inventio ces nouvelles machines, élémen fondateur science informatique est 1936, publication articl "on computable numbers with application the entscheidungsproblem"[10] par alan turing qui allait déplacer centre préoccupation certains scientifiques (mathématiciens logiciens) époque sujet calculabilité (ou décidabilité) ouvert par hilbert, malmené par godël, éclairci par church, vers sujet mécanisation calcul (ou calculabilité effective). dans texte pages, turing expose une machine théorique capable effectue tout calcul ; démontre que cette machine est aussi puissante, niveau calcul, que tout être humain. autrement dit, problème mathématique possède une solution, seulement si, existe une machine turing capable résoudre problème. par suite, expose une machine turing universelle apte reproduire toute machine turing, agi des concepts ordinateur programmation programme. termine démontrant existe moins problème mathématique formellement insoluble, problème arrêt peu avant seconde guerre mondiale, apparurent les premières calculatrices électromécaniques, construites selon les idées ala turing. les machines furent vite supplantées par les premiers calculateurs électroniques, nettement plus performants. [modifier] seconde guerre mondiale premier ordinateur fonctionnant langage binaire fut colossus, conçu lors seconde guerre mondiale, étai pas turing-complet bien ala turing ait travaillé projet. fin guerre, fut démonté caché cause son importance stratégique. juin 1945 est publié article fondateur john [11] donnant les bases architectur utilisée dans quasi totalité des ordinateurs depuis lors. dans cet article von neumann veut concevoir programme enregistré programmé dans machine. première machine correspondant cette architecture, dite depuis architecture von neumann est une machine expérimentale small-scale experimental machine (ssem "baby") construite manchester juillet 1948. aout 1949 première machine fonctionnelle, fondé sur les bases von neumann fut edvac eniac mis service 1946, est premier ordinateur entièrement électronique construit pour être turing-complet. [modifier] années 1950 mot ordinateur fut introduit par ibm france 1955. françois girard, alors responsable service publicité entreprise eut idé consulter son ancien professeur lettres paris, jacques perret, afin lui demander proposer mot caractérisant mieux possible que appelait vulgairement calculateur (traduction littérale mot anglais « computer »). dernier proposa « ordinateur », mot tombé désuétude désignant anciennement ordonnateur, voire notion ordr ecclésiastique dans églis catholique (ordinant)[12],[13]. professeur suggéra plus précisément « ordinatrice électronique », féminin ayant permettre, selon lui, mieux distinguer usag religieux usag comptable mot[14],[15]. premier ordinateur multitâches est bull gamma 1958[réf. nécessaire]. [modifier] premier ordinateur familial premier ordinateur dit familial est appl ii, lancé janvier 1977. été suivi nombreux autres, notamment atar 400 1979, ti99/4a, hecto sinclair zx81 1981, les msx, commodore 1982, oric- 1983, appl iie, amstra cpc 464 1984 (suivi des 664 puis 6128)[16]. [modifier] généralités les ordinateurs furent abor utilisés pour calcul (en nombres entiers abord puis flottants). peut cependant les assimiler simples calculateurs : effet, résultat traitement ordinateur peut être non seulement une série nombres, mais aussi nouveau programme (utilisable par cet ordinateur par autre). dans architectur von neumann, les données sont banalisées peuvent être interprétées indifféremment comme des nombres, des instructions, des valeurs logiques tout symbole défini arbitrairement (lettre l’alphabet, par exemple). calcul représente une des applications possibles. dans cas, les données sont traitées comme des nombres. l’ordinateur est utilisé aussi pour ses possibilités organisatio l’information, entre autres sur des périphériques stockage magnétique. calculé fin des années 1980 que sans les ordinateurs faudrait toute population française juste pour faire dans pays seul travail des banques. cette capacité d’organiser les informations généralisé l’usage traitement texte dans grand public ; gestion des bases données relationnelles permet également retrouver consolider des informations réparties vues par utilisateu comme plusieurs tables indépendantes. cette création néologisme fut origin traductions multiples des expressions supercomputer, superordinateur supercalculateur, quantum computer, calculateur quantique ordinateur quantique. dans dernier cas, utilisatio mot « ordinateur » est justement surfaite car les possibilités envisageables pour calcul quantique sont loin polyvalence « ordinateur ». l’expérience appris distinguer dans ordinateur deux aspects, dont second avait été départ sous-estimé : l’architecture physique, matérielle (alias hardware hard) ; l’architecture logicielle (alias software soft) ; ordinateur très avancé techniquement pour son époque comme gamma compagnie bull n’eut pas succès attendu, pour simple raison qu’il existait peu moyens mettre œuvre commodément ses possibilités techniques. logiciel son complément les services (formation, maintenance, ...) forme depuis milieu des années 1980 l’essentiel des coûts d’équipement informatique, matériel n’y ayant qu’une part minoritaire. les ordinateurs peuvent être sensibles aux bombes iem. [modifier] fonctionnement vue ensembl des différents organes ordinateur parmi toutes les machines inventées par homme ordinateu est celle qui rapproche plus concept anthropologique suivant : organe entrée organe traitement informatio organe sortie. chez homme les organes entré sont les cinq sens, organ traitement est cerveau dont les logiciels sont apprentissag avec des mises jour constantes cours vie, puis les organes sortie sont les muscles. pour les ordinateurs modernes, les organes entré sont clavier souris les organes sortie, écran imprimante graveur dvd, etc.] les techniques utilisées pour fabriquer ces machines ont énormément changé depuis les années 1940 sont devenues une technologie (c’est-à-dire ensemble industriel organisé autour techniques) part entière depuis les années 1970. beaucoup utilisent encore les concepts définis par john von neumann, bien que cette architecture soit régression : les programmes modifient plus guère eux-mêmes (ce qui serait considéré comme une mauvaise pratique programmation), matériel prend compte cette nouvelle donne séparant nettement stockage des instructions des données, compris dans les caches. l’architecture von neumann décomposait l’ordinateur quatre parties distinctes : l’unité arithmétique logique (ual) unité traitement : son rôle est d’effectuer les opérations base, peu comme ferait une calculatrice ; l’unité contrôle. c’est l’équivalent des doigts qui actionneraient calculatrice ; mémoire qui contient fois les données programme qui dira l’unité contrôle quels calculs faire sur ces données. mémoire divise entre mémoire vive (programmes données cours fonctionnement) mémoire permanente (programmes données base machine) ; les entrées-sorties : dispositifs qui permettent communiquer avec monde extérieur. [modifier] ual l’unité arithmétique logique ual est l’élément qui réalise les opérations élémentaires (additions, soustractions...), les opérateurs logiques (et, ou, ni, etc.) les opérations comparaison (par exemple comparaison d’égalité entre deux zones mémoire). c’est l’ual qui effectue les calculs l’ordinateur. l’unité contrôle prend ses instructions dans mémoire. celles-ci lui indiquent qu’elle doit ordonner l’ual et, comment elle devra éventuellement agir selon les résultats que celle-ci lui fournira. une fois l’opération terminée, l’unité contrôle passe soit l’instruction suivante, soit une autre instruction laquelle programme lui ordonne brancher. unit contrôle facilite communication entre unit arithmétique logique, mémoire ainsi que les périphériques. elle gère plupart des exécutions des instructions dans ordinateur [modifier] mémoire sein système, mémoire peut être décrite comme une suite cellules numérotées contenant chacune une petite quantité d’informations. cette information peut servir indiquer l’ordinateur qu’il doit faire (instructions) contenir des données traiter. dans plupart des architectures, même mémoire qui est utilisée pour les deux fonctions. dans les calculateurs massivement parallèles, admet même que des instructions programmes soient substituées d’autres cours d’opération lorsque cela traduit par une plus grande efficacité. cette pratique était jadis courante, mais les impératifs lisibilité génie logiciel fait régresser, hormis dans cas particulier, depuis plusieurs décennies. cette mémoire peut être réécrite autant fois que nécessaire. taille chacun des blocs mémoire ainsi que technologie utilisée ont varié selon les coûts les besoins : bits pour les télécommunications, bits pour l’instrumentation (dec) bits pour gros calculateurs scientifiques (control data). consensus fini par être trouvé autour l’octet comme unité adressable d’instructions sur format 8 octets. dans tous les cas figure, octe reste adressable, qui simplifie écritur des programmes. les techniques utilisées pour réalisation des mémoires ont compris des relais électromécaniques, des tubes mercure sein desquels étaient générées des ondes acoustiques, des transistors individuels, des tores ferrite enfin des circuits intégrés incluant des millions transistors. [modifier] entrées-sorties les dispositifs d’entrée/sortie permettent l’ordinateur communiquer avec l’extérieur. ces dispositifs sont très importants, clavier l’écran. carte réseau permet par exemple relier les ordinateurs réseau informatique, dont plus grand est internet. point commun entre tous les périphériques d’entrée est qu’ils convertissent l’information qu’ils récupèrent l’extérieur données compréhensibles par l’ordinateur. l’inverse, les périphériques sortie décodent l’information fournie par l’ordinateur afin rendre compréhensible par l’utilisateur. [modifier] bus ces différentes parties sont reliées par trois bus, bus adresse bus données bus commande. bus est groupement certain nombre fils électriques réalisant une liaison pour transporter des informations binaires codées sur plusieurs bits. bus adress transporte les adresses générées par (unité centrale traitement) pour sélectionner une case mémoire registre interne des blocs. nombre bits véhiculés par bus dépend quantité mémoire qui doit être adressée. bus données transporte les données échangées entre les différents éléments système. bus contrôle transporte les différents signaux synchronisation nécessaires fonctionnement système : signal lecture (rd), signal écritur (wr), signal sélection (cs : chip select). [modifier] architecture miniaturisation permet d’intégrer l’ual l’unité contrôle sein d’un même circuit intégré connu sous nom microprocesseur. typiquement, mémoire est située sur des circuits intégrés proches processeur, une partie cette mémoire, mémoire cache, pouvant être située sur même circuit intégré que l’ual. l’ensemble est, sur plupart des architectures, complété d’une horloge qui cadence processeur. bien sûr, souhaite ell soit plus rapide possible, mais peut pas augmenter sans limites vitesse pour deux raisons : plus l’horloge est rapide plus processeur dégage chaleur (selon carré fréquence). une trop grande température peut détériorer processeur ; existe une cadence processeur devient instable ; génère des erreurs qui mènent plus souvent plantage. tendance été partir 2004 regrouper plusieurs ual dans même processeur, voire plusieurs processeurs dans même puce. effet, miniaturisation progressive (voir loi moore) permet sans grand changement coût. une autre tendance, depuis 2006 chez arm, est aux microprocesseurs sans horloge : moitié dissipation thermique est effet due aux signaux horlog quand microprocesseur fonctionne ; plus, microprocesseur sans horloge une consommation presque nulle quand fonctionne pas : seul signal horlog nécessaire est alors celui destiné rafraîchissement des mémoires. cet atout est important pour les modèles portables. principal écart fonctionnel aujourd’hui par rapport modèle von neumann est présence sur certaines architectures deux antémémoires différentes : une pour les instructions une pour les données (alors que modèle von neumann spécifiait une mémoire commune pour les deux). raison cet écart est que modification par programme ses propres instructions est aujourd’hui considérée (sauf sur les machines hautement parallèles) comme une pratique proscrire. dès lors, contenu cache données doit être récrit mémoire principale quand est modifié, sait que celui cache d’instructions n’aura jamais l’être, d’où simplification des circuits gain performance. [modifier] instructions les instructions que l’ordinateur peut comprendre sont pas celles langage humain. matériel sait juste exécuter nombre limité d’instructions bien définies. des instructions typiques comprises par ordinateur sont « copier contenu cellule 123 placer dans cellule 456 », « ajouter contenu cellule 321 celui cellule 654 placer résultat dans cellule 777 » « si contenu cellule 999 vaut exécuter l’instruction cellule 345 ». mais plupart des instructions composent deux zones : l’une indiquant quoi faire, nommée code opération, l’autre indiquant faire, nommée opérande. sein l’ordinateur, les instructions correspondent des codes code pour une copie étant par exemple 001. l’ensemble d’instructions qu’un ordinateur supporte nomme son langage machine, langage qui est une succession chiffres binaires, car les instructions données qui sont comprises par processeur (cpu) sont constituées uniquement (zéro) (un). courant électrique passe pas. courant électrique passe. général, les programmeurs n’utilisent plus type langage, mais passent par que l’on appelle langage haut niveau qui est ensuite transformé langage binaire par programme spécial (interpréteur compilateur selon les besoins). les programmes ainsi obtenus sont des programmes compilés compréhensibles par ordinateu dans son langage natif. certains langages programmation, comme l’assembleur sont dits langages bas niveau car les instructions qu’ils utilisent sont très proches celles l’ordinateur. les programmes écrits dans ces langages sont ainsi très dépendants plate-forme pour laquelle ils ont été développés. langage beaucoup plus facile relire que l’assembleur, permet donc aux programmeurs d’être plus productifs. pour cette raison, l’a plus plus utilisé mesure que les coûts matériel diminuaient que les salaires horaires des programmeurs augmentaient. [modifier] logiciels les logiciels informatiques sont longues listes d’instructions exécutables par ordinateur. nombreux programmes contiennent des millions d’instructions, effectuées pour certaines manière répétitive. nos jours, ordinateur personnel exécute plusieurs milliards d’instructions par seconde. depuis milieu des années 1960, des ordinateurs exécutent plusieurs programmes simultanément. cette possibilité est appelée multitâche. c’est cas tous les ordinateurs aujourd’hui[quand ?]. réalité, chaque cœur processeur n’exécute qu’un programme fois, passant d’un programme l’autre chaque fois que nécessaire. rapidité processeur est suffisamment grande par rapport nombre tâches exécuter, l’utilisateur aura l’impression d’une exécution simultanée des programmes. les priorités associées aux différents programmes sont, général, gérées par système exploitation [modifier] système d’exploitation article détaillé : système exploitation système d’exploitation est programme central qui contient les programmes base nécessaires bon fonctionnement des applications l’ordinateur. système d’exploitation alloue les ressources physiques l’ordinateur (temps processeur, mémoire...) aux différents programmes cours d’exécution. fournit aussi des outils aux logiciels (comme les pilotes) afin leur faciliter l’utilisation des différents périphériques sans avoir connaître les détails physiques. électronique qui fonctionne par lecture séquentielle ensemble instruction qui lui font exécuter des opérations logiques arithmétiques sur des chiffres binaires. dès mise sous tension, ordinateur exécute, après autre des instructions qui lui font lire, manipuler, puis réécrire ensemble données auquel accès. des tests des sauts conditionnels permettent changer instructio suivante, donc agi différemment fonction des données des nécessités moment. les données manipuler sont obtenues, soit par lecture mémoires, soit par lecture composants interfac (périphériques) qui représentent des données physiques extérieures valeurs binaires (déplacement souris, touche appuyée sur clavier, température, vitesse, compression...). une fois utilisées, manipulées, les données sont réécrites, soit dans des mémoires, soit dans des composants qui peuvent transformer une valeur binaire une action physique (écriture sur une imprimante sur moniteur, accélération freinage véhicule, changement température four ...). ordinateu peut aussi répondre des interruptions qui lui permettent d’exécuter des programmes réponses spécifiques chacune, puis reprendre l’exécution séquentielle programme interrompu. technique actuelle des ordinateurs date milieu xxe siècle. ils peuvent être classés selon plusieurs critères[1] (domaine application taille architecture). éclaté ordinateur personnel : [modifier] machines calculer les machines calculer jouèrent rôle primordial dans développement des ordinateurs pour deux raisons tout fait indépendantes. part, pour leurs origines : pendant développement machine calculer automatique imprimante 1834 charles babbage commença imaginer machine analytique, l’ancêtre des ordinateurs. c’était une machine calculer programmée par lecture cartes perforées (inspirées métier jacquard), avec lecteur cartes pour les données pour les programmes, avec des mémoires, calculateur central des imprimantes qui inspirera développement des premiers ordinateurs cent ans plus tard (howard aiken fit beaucoup référence aux machines babbage quand convainquit ibm construire harvar mark 1937) ; qui nous amènera aux mainframes des années 1960. autr part, leur propagation fit grâce commercialisation 1971 premier microprocesseur, inte 4004, qui fut inventé pendant développement machine calculer électronique pour compagnie japonaise busicom, qui est origin explosio micro-informatique partir 1975[8] qui réside cœur tous les ordinateurs actuels quel que soit leurs tailles fonctions (bien que seulement 2 % des microprocesseurs produits chaque années soient utilisés comme unités centrales ordinateur les 98 % restant sont utilisés dans construction voitures, robots ménager, montres, caméras surveillance[9]...). [modifier] électromécanique mécanographie outre les avancées observées dans industri textile celles électronique les avancées mécanographie fin xixe siècle, pour achever les recensements aux états-unis amérique mécanisation cryptographie début xxe siècle, pour chiffrer puis déchiffrer automatiquement des messages, développement des réseaux téléphoniques base relais électromécaniques), sont aussi prendre compte pour comprendre avènemen nouveau genre machine qui calculent pas (comme font/faisaient les calculatrices), mais lisent interprètent des programmes qui -eux- calculent. pour monde des idées, avant inventio ces nouvelles machines, élémen fondateur science informatique est 1936, publication articl "on computable numbers with application the entscheidungsproblem"[10] par alan turing qui allait déplacer centre préoccupation certains scientifiques (mathématiciens logiciens) époque sujet calculabilité (ou décidabilité) ouvert par hilbert, malmené par godël, éclairci par church, vers sujet mécanisation calcul (ou calculabilité effective). dans texte pages, turing expose une machine théorique capable effectue tout calcul ; démontre que cette machine est aussi puissante, niveau calcul, que tout être humain. autrement dit, problème mathématique possède une solution, seulement si, existe une machine turing capable résoudre problème. par suite, expose une machine turing universelle apte reproduire toute machine turing, agi des concepts ordinateur programmation programme. termine démontrant existe moins problème mathématique formellement insoluble, problème arrêt peu avant seconde guerre mondiale, apparurent les premières calculatrices électromécaniques, construites selon les idées ala turing. les machines furent vite supplantées par les premiers calculateurs électroniques, nettement plus performants.
